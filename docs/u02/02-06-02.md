# 2. Compatibilidad

Cada navegador (Chrome, Firefox, Edge, Safari, etc.) interpreta el código HTML siguiendo los estándares, pero **pueden existir pequeñas diferencias** en cómo se muestran las páginas.  
Por ello es importante realizar **pruebas de compatibilidad** en varios navegadores.

---

## Diferencias entre navegadores

- **Motores de renderizado distintos:**  
  - Chrome / Edge → *Blink*  
  - Firefox → *Gecko*  
  - Safari → *WebKit*  
- Pequeñas variaciones en:
  - Tipografía por defecto.  
  - Espaciados y márgenes iniciales.  
  - Interpretación de atributos obsoletos.  
  - Reproducción de multimedia.  

---

## Herramientas de prueba

- **Navegadores instalados en el PC:** abrir el mismo archivo en distintos navegadores.  
- **Modo desarrollador (DevTools):** permite inspeccionar elementos y probar cambios en tiempo real.  
- **Pruebas en dispositivos móviles:** Chrome y Firefox permiten simular móviles y tablets en DevTools.  
- **Servicios online:** BrowserStack, LambdaTest (simulan distintos navegadores y sistemas operativos).  

---

## Estrategias recomendadas

1. **Probar en al menos dos navegadores principales:** Chrome y Firefox.  
2. **Verificar en dispositivos móviles:** comprobar el diseño responsive (más adelante con CSS).  
3. **Comprobar fuentes y caracteres especiales:** a veces varían entre navegadores.  
4. **Analizar el soporte de etiquetas modernas:** algunos navegadores antiguos no soportan elementos HTML5.  

---

## Ejemplo práctico: inspección en Chrome DevTools

1. Abre la página en Chrome.  
2. Pulsa `F12` o clic derecho → **Inspeccionar**.  
3. Navega por el árbol del DOM.  
4. Cambia atributos directamente para ver cómo afectan.  
5. Activa la opción de **vista móvil** para comprobar el diseño en pantallas pequeñas.
