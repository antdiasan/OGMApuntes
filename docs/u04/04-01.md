# El concepto de layout

En el apartado anterior hemos visto qué es la **maquetación web**.  
Ahora vamos a centrarnos en uno de los conceptos más importantes dentro de la maquetación: el **layout**.

Entender bien qué es un layout es fundamental para comprender por qué los elementos se colocan de una determinada forma en una página web.

---

## ¿Qué es un layout?

En desarrollo web, el término **layout** hace referencia a la **organización y distribución de los elementos dentro de una página web**.

> **Definición**  
> Un **layout** es la forma en la que se colocan y distribuyen los elementos de una página web en la pantalla.

El layout define:

- qué elementos van arriba o abajo  
- qué elementos se colocan uno al lado del otro  
- cómo se reparte el espacio disponible  
- el orden visual de los bloques  

---

## Layout y estructura visual

El layout puede entenderse como el **esqueleto visual** de una página web.

No define colores, tipografías ni estilos gráficos, sino **la estructura básica** sobre la que se construye la página.

> **Idea clave**  
> El layout indica **dónde va cada elemento**, no cómo se ve.

---

## Ejemplo de layout sencillo

```text
-------------------------
|       CABECERA        |
-------------------------
|      CONTENIDO        |
-------------------------
|     PIE DE PÁGINA     |
-------------------------
```

En este layout:

- los elementos están organizados en una sola columna  
- cada bloque ocupa toda la anchura disponible  
- los bloques se colocan uno debajo del otro  

---

## Layout de varias columnas

```text
-------------------------
|       CABECERA        |
-------------------------
| MENÚ | CONTENIDO      |
|      |                |
-------------------------
|     PIE DE PÁGINA     |
-------------------------
```

En este caso:

- los elementos no solo se colocan en vertical  
- también se distribuyen en horizontal  
- el espacio se reparte entre distintas zonas  

---

## Formas habituales de diseñar un layout

Además de la estructura visual, un layout puede **comportarse de distintas formas según el tamaño de la pantalla**.

### Layout fijo

- ancho fijo  
- no se adapta a la pantalla  
- común en webs antiguas  

---

### Layout fluido

- usa porcentajes  
- se adapta parcialmente  
- mejora la flexibilidad  

---

### Layout responsive

- se adapta a cada dispositivo  
- cambia su distribución  
- es el enfoque actual  

> Hoy en día, cuando hablamos de maquetación web, normalmente hablamos de **layouts responsive**.

---

### Límites de tamaño

En muchos layouts se usan límites como:

- `max-width`
- `min-width`

para evitar diseños incómodos en pantallas muy grandes o pequeñas.

---

## Layout y CSS

El layout se construye principalmente con **CSS**:

- flujo normal  
- `display`  
- modelo de caja  
- centrado  
- posicionamiento  
- Flexbox  

> Antes de usar sistemas modernos de layout, es importante entender el comportamiento por defecto de los elementos.

---

## Layout y responsive

Un buen layout debe ser:

- flexible  
- adaptable  
- pensado para móvil y escritorio
