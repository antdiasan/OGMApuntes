# Maquetación con inline-block y columnas

Antes de utilizar herramientas modernas de maquetación como Flexbox, una de las técnicas más habituales para crear **columnas** en CSS era el uso de la propiedad `display: inline-block`.

Aunque hoy en día no es la técnica más recomendable para layouts complejos, es importante conocerla porque:

  - ayuda a entender cómo funciona el flujo del documento,
  - permite comprender mejor los problemas de alineación y espacios,
  - y sigue apareciendo en proyectos antiguos.

---

## El objetivo de la maquetación en columnas

La maquetación en columnas consiste en **colocar varios bloques uno al lado del otro**, repartiendo el espacio horizontal disponible.

Por ejemplo:

  - un menú lateral y un contenido principal,
  - varias tarjetas alineadas en una fila,
  - bloques de información en columnas.

---

## Uso de inline-block para crear columnas

Cuando un elemento tiene `display: inline-block`:

  - se comporta como un elemento en línea,
  - se coloca junto a otros elementos,
  - pero mantiene comportamiento de caja (ancho y alto).

Esto permite crear columnas sin que los elementos bajen automáticamente a la siguiente línea.

```css
.columna {
  display: inline-block;
  width: 30%;
}
```

---

## Ejemplo básico de columnas con inline-block

Supongamos el siguiente HTML:

```html
<div class="columna">Columna 1</div>
<div class="columna">Columna 2</div>
<div class="columna">Columna 3</div>
```

Y el siguiente CSS:

```css
.columna {
  display: inline-block;
  width: 30%;
  padding: 10px;
  border: 1px solid black;
  vertical-align: top;
}
```

Las tres columnas se colocan una junto a la otra dentro del ancho disponible.

---

## El problema de los espacios entre columnas

Al usar `inline-block`, el navegador interpreta los **espacios y saltos de línea del HTML** como espacios reales.

Esto puede provocar:

  - huecos inesperados entre columnas,
  - que una columna no quepa y baje a la siguiente línea.

Este comportamiento no es un error, sino una consecuencia de tratar los elementos como contenido en línea.

---

## Soluciones habituales al espacio entre inline-block

Algunas soluciones tradicionales para este problema han sido:

  - eliminar los saltos de línea en el HTML,
  - reducir el tamaño total de las columnas,
  - usar comentarios HTML para evitar espacios,
  - ajustar el tamaño de fuente del contenedor.

Estas soluciones funcionan, pero no son especialmente limpias ni cómodas.

---

## Limitaciones de inline-block para maquetar

Aunque `inline-block` permite crear columnas, presenta varias limitaciones:

  - gestión complicada de espacios,
  - alineación vertical poco intuitiva,
  - dificultad para layouts flexibles,
  - poca adaptación a distintos tamaños de pantalla.

Por este motivo, hoy en día se considera una técnica **transitoria**.

---

## Relación con técnicas modernas

El uso de `inline-block` ayuda a comprender:

  - por qué surgieron nuevas herramientas de maquetación,
  - qué problemas resuelven,
  - y por qué Flexbox es más adecuado para layouts actuales.
