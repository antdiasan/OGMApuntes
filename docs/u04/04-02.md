# El flujo de los elementos en el navegador

Antes de empezar a colocar elementos con técnicas de maquetación más avanzadas, es fundamental entender cómo el navegador **coloca los elementos por defecto**.

Este comportamiento inicial se conoce como **flujo normal del documento** y es la base de todo el layout en CSS.

---

## ¿Qué es el flujo del documento?

El **flujo del documento** es la forma en la que el navegador coloca los elementos HTML en la página **cuando no se aplica ninguna regla de maquetación especial**.

!!! note "Definición"
    El flujo del documento es el comportamiento por defecto que sigue el navegador para mostrar los elementos HTML en una página web.

---

## Cómo coloca el navegador los elementos

Cuando no aplicamos CSS de maquetación, el navegador sigue estas reglas básicas:

- Los elementos **en bloque** se colocan **uno debajo de otro**.
- Los elementos **en línea** se colocan **uno al lado del otro**.
- Los elementos se muestran **en el orden en el que aparecen en el HTML**.

Este comportamiento es automático y no requiere ninguna propiedad CSS adicional.

---

## Elementos en bloque en el flujo normal

Los elementos en bloque:

- Empiezan siempre en una nueva línea.
- Ocupan toda la anchura disponible.
- Se colocan uno debajo del otro.

Ejemplos habituales de elementos en bloque son:

- `<div>`
- `<p>`
- `<h1>` a `<h6>`
- `<section>`

```html
<h1>Título</h1>
<p>Párrafo 1</p>
<p>Párrafo 2</p>
```

En este caso, cada elemento aparece en una línea diferente.

---

## Elementos en línea en el flujo normal

Los elementos en línea:

- No empiezan en una nueva línea.
- Se colocan junto a otros elementos en línea.
- Ocupan solo el espacio que necesita su contenido.

Ejemplos habituales de elementos en línea son:

- `<span>`
- `<a>`
- `<strong>`
- `<em>`

```html
<p>
  Este texto tiene un <span>elemento en línea</span> dentro del párrafo.
</p>
```

---

## Importancia del orden en el HTML

En el flujo normal, el **orden del HTML es muy importante**.

!!! tip "Idea clave"
    El navegador muestra los elementos siguiendo exactamente el orden en el que aparecen en el código HTML.

Si cambiamos el orden de los elementos en el HTML, cambia también su posición en la página.

---

## El flujo normal y la maquetación

Antes de usar propiedades como `display`, `position` o Flexbox, el navegador siempre parte del **flujo normal**.

Por eso es importante:

- Entender cómo se comportan los elementos por defecto.
- Saber por qué los elementos aparecen arriba, abajo o en línea.
- Comprender que muchas propiedades CSS modifican este flujo.

!!! note "Importante"
    La maquetación en CSS consiste, en gran parte, en **modificar el flujo normal del documento**.

---

## Qué NO hace el flujo normal

El flujo normal **no**:

- Centra elementos automáticamente.
- Coloca elementos en columnas complejas.
- Superpone elementos.
- Adapta el diseño al tamaño de pantalla.

Estas tareas requieren el uso de propiedades CSS adicionales.
