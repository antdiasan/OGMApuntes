# La propiedad position en CSS

La propiedad `position` permite modificar la forma en la que un elemento se coloca en la página web.
Según el valor que se utilice, el elemento puede seguir el flujo normal del documento o salirse de él.

Además, `position` permite utilizar propiedades como `top`, `bottom`, `left`, `right` y `z-index`.

---

## position: static

Es el **valor por defecto** de todos los elementos HTML.

- El elemento sigue el flujo normal del documento.
- Las propiedades `top`, `bottom`, `left`, `right` y `z-index` **no tienen efecto**.

```css
.caja {
  position: static;
}
```

En la práctica, no suele declararse explícitamente.

---

## position: relative

El elemento mantiene su posición en el flujo normal, pero puede desplazarse respecto a su posición original.

- El espacio original del elemento **se mantiene**.
- Permite usar `top`, `bottom`, `left` y `right`.
- Sirve como referencia para elementos con `position: absolute`.

```css
.caja {
  position: relative;
  top: 20px;
  left: 10px;
}
```

---

## position: absolute

El elemento **sale del flujo normal del documento**.

- No reserva espacio.
- Se posiciona respecto al **primer elemento padre con `position: relative`**.
- Si no existe, se posiciona respecto al documento.

```css
.caja {
  position: absolute;
  top: 50px;
  left: 100px;
}
```

Es muy utilizado para superponer elementos.

---

## position: fixed

El elemento se posiciona respecto a la **ventana del navegador**.

- No se mueve al hacer scroll.
- Sale del flujo normal.
- Muy usado para menús o botones flotantes.

```css
.caja {
  position: fixed;
  bottom: 20px;
  right: 20px;
}
```

---

## position: sticky

Combina el comportamiento de `relative` y `fixed`.

- Se comporta como `relative` hasta que se alcanza una posición de scroll.
- A partir de ese momento, se comporta como `fixed`.

```css
.caja {
  position: sticky;
  top: 0;
}
```

Es necesario definir al menos una propiedad de posición (`top`, `bottom`, etc.).

---

## Otros valores de position

### inherit

El valor `inherit` hace que el elemento herede el valor de `position` de su elemento padre.

```css
.caja {
  position: inherit;
}
```

No es un valor de uso habitual y se incluye como **cultura general**.

---

## La propiedad z-index

La propiedad `z-index` controla el **orden de superposición** de los elementos.

- Solo funciona en elementos con `position` distinto de `static`.
- Cuanto mayor es el valor, más arriba se sitúa el elemento.

```css
.caja1 {
  position: absolute;
  z-index: 1;
}

.caja2 {
  position: absolute;
  z-index: 10;
}
```

El elemento con mayor `z-index` se mostrará por encima del resto.

> ⚠️ Nota: no es necesario profundizar en casos avanzados; para la mayoría de situaciones basta con usar valores simples.
